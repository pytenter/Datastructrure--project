# 项目报告模板（新能源物流车队协同调度）

## 1. 项目背景与目标
- 业务背景：中央仓库 + 新能源物流车队 + 城市动态任务。
- 目标：在电量、载重、时间窗和充电站拥塞约束下提升总收益。

## 2. 问题定义
- 道路网络：无向加权图 `G=(V,E)`。
- 车辆集合：`K`，每车有 `battery_capacity, capacity, speed`。
- 任务集合：动态释放，任务属性 `(release_time, location, weight, deadline)`。
- 充电站集合：每站有 `ports, charge_rate`，存在排队等待。

## 3. 数据结构与系统设计
- 图结构：邻接表 + Dijkstra 最短路。
- 事件驱动仿真：按时间推进，任务动态进入待分配池。
- 站点排队：用端口“最早可用时间”模拟队列和负载。

## 4. 调度策略
- 策略 A：最近任务优先（`nearest_task_first`）。
- 策略 B：最大任务优先（`max_task_first`）。
- 可选：多车协同开关（超重任务可拆分为多车执行）。

## 5. 评分函数
- 任务得分与响应时间、路径长度负相关。
- 超时任务扣分，未服务任务固定惩罚。
- 给出公式与参数取值，并解释参数含义。

## 6. 实验设置
- 三种规模：`small / medium / large`。
- 随机种子：保证可复现。
- 指标：完成数、未完成数、超时数、总里程、平均响应时间、充电等待、总分。

## 7. 实验结果与分析
- 表格对比两种策略在三种规模下的表现。
- 分析差异原因：任务优先级、路径绕行、充电拥塞、局部贪心行为。
- 指出在大规模场景下的瓶颈与改进方向。

## 8. 进阶扩展（可加分）
- 动态策略 vs 静态全局最优（上帝视角）对比。
- 引入 RL / 元启发式 / 多智能体方法。
- 图形界面可视化（地图、轨迹、队列、分数曲线）。

## 9. 小组分工
- 成员 A：图建模与仿真引擎。
- 成员 B：调度策略与实验设计。
- 成员 C：结果分析与可视化/视频。
- 成员 D（可选）：进阶算法与优化器对接。

- 总结系统工程价值与后续可落地方向。
